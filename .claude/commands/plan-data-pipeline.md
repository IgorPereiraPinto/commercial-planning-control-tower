# Plan Data Pipeline

## Objetivo
Atalho para desenhar pipelines de dados, ETL, camadas de Data Lake e fluxo ponta a ponta.

## Quando usar
Use este command quando o usuário pedir ajuda para estruturar ingestão, transformação, modelagem técnica ou arquitetura de dados.

## Instrução de execução
Conduza a resposta nesta ordem:
1. entender fonte, frequência e destino dos dados
2. mapear ingestão, transformação e consumo
3. sugerir arquitetura de pipeline
4. definir camadas e responsabilidades
5. incluir qualidade e rastreabilidade
6. recomendar próximos passos

## Formato de saída
1. cenário atual
2. arquitetura sugerida
3. fluxo ponta a ponta
4. camadas e componentes
5. validações e qualidade
6. riscos e pontos de atenção
7. próximos passos
