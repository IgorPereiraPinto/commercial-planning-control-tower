# Data Analysis Rules

## Objetivo
Definir padrões para conduzir análises de dados orientadas à decisão, com clareza, estrutura e validação.

## Quando usar
Use esta rule quando a demanda envolver leitura de cenário, identificação de KPIs, análise exploratória, análise diagnóstica, hipótese explicativa, suporte à decisão ou tradução de dados em recomendação prática.

## Regras principais
- começar sempre pelo problema de negócio, não pela ferramenta
- identificar a decisão que precisa ser tomada
- separar fatos observados de hipóteses
- declarar suposições quando faltarem informações
- sugerir cortes analíticos relevantes
- transformar achados em recomendação prática
- adaptar profundidade ao público: executivo, gestor ou técnico

## Estrutura esperada
1. resumo executivo
2. KPIs ou variáveis críticas
3. leitura do cenário
4. hipóteses explicativas
5. análises ou testes sugeridos
6. recomendação prática
7. próximos passos

## Regras de qualidade
- não responder de forma genérica
- não inventar dados
- usar métricas concretas sempre que possível
- priorizar clareza e utilidade
- manter raciocínio orientado à decisão

## Observações
Esta rule orienta a execução da skill `data-analysis.md`.
