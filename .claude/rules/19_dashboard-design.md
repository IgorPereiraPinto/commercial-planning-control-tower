# Dashboard Design Rules

## Objetivo
Definir padrões para desenho conceitual de dashboards, escolha de visuais, layout, hierarquia e comunicação visual orientada à decisão.

## Quando usar
Use esta rule quando a demanda envolver escolha de gráfico, estrutura de painel, distribuição de componentes, organização de filtros, layout gerencial ou comunicação visual de dados sem necessariamente gerar código.

## Regras principais
- começar pela pergunta de negócio, não pelo gráfico
- escolher visual pelo que ele responde
- priorizar hierarquia visual clara
- organizar o painel para leitura rápida
- manter coerência entre filtros, KPIs e visuais
- considerar público-alvo e tipo de decisão
- usar cor como reforço analítico, não decoração
- evitar poluição visual e excesso de elementos

## Estrutura esperada
1. objetivo do painel
2. público e decisão esperada
3. estrutura visual sugerida
4. visuais recomendados
5. sequência de leitura
6. pontos de atenção
7. próximos passos

## Regras de qualidade
- não sugerir gráfico sem justificativa
- não usar visuais inadequados para comparação, ranking ou tendência
- não criar dashboard decorativo
- sempre conectar layout à narrativa de negócio
- priorizar clareza, foco e utilidade
