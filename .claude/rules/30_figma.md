# Figma Rules

## Objetivo
Definir padrões para uso do Figma em wireframes, protótipos e interfaces analíticas com foco em clareza, hierarquia visual e UX.

## Quando usar
Use esta rule quando a demanda envolver wireframe, mockup, prototipação, organização de dashboard, estrutura de tela, design system leve ou handoff visual no Figma.

## Regras principais
- começar pelo objetivo da interface, não pela estética
- identificar o público e a ação ou decisão esperada
- organizar a tela com hierarquia visual clara
- separar visual principal, suporte e detalhe
- usar componentes reutilizáveis quando fizer sentido
- priorizar consistência de espaçamento, alinhamento e tipografia
- usar cor como reforço visual e analítico
- diferenciar wireframe, mockup e protótipo conforme o nível de maturidade

## Estrutura esperada
1. objetivo da tela
2. público e contexto
3. layout sugerido
4. componentes principais
5. hierarquia visual
6. observações de UX
7. próximos passos

## Regras de qualidade
- não desenhar interface apenas para ficar bonita
- não exagerar em elementos decorativos
- não misturar muita informação sem hierarquia
- priorizar leitura rápida e clareza
- manter consistência entre telas do mesmo fluxo

## Observações
Esta rule orienta a execução da skill `figma.md`.
