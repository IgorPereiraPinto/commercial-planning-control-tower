# Dashboard HTML Rules

## Objetivo
Definir padrões para criação de dashboards web analíticos com HTML, CSS, JavaScript e Chart.js, com foco em UX executivo e estrutura pronta para uso.

## Quando usar
Use esta rule quando a demanda envolver dashboard web, painel HTML, interface analítica para portfólio, adaptação de Power BI para web, ou dashboard single-file com filtros e KPIs.

## Regras principais
- entender primeiro a pergunta de negócio
- definir público, KPIs e filtros antes do código
- separar camada de dados, cálculos, filtros, narrativa e renderização
- priorizar layout executivo e responsivo
- preferir single-file quando fizer sentido para portfólio e demonstração
- preparar a estrutura para futura troca por dados reais
- comentar as funções principais
- manter o dashboard analítico, não apenas visualmente bonito

## Estrutura esperada
1. resumo da arquitetura
2. estrutura visual
3. código HTML/CSS/JS
4. lógica dos filtros
5. narrativa executiva
6. como adaptar para dados reais
7. observações de manutenção

## Regras de qualidade
- não criar filtros decorativos
- não deixar texto executivo desconectado dos dados
- não misturar tudo em uma única função sem organização
- evitar visual genérico ou poluído
- priorizar clareza, navegabilidade e reutilização

## Observações
